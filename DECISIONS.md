# CredStack — Decision Log

Locked decisions. Override only with explicit reason recorded here.

## Product

- **Community features (9.8 forums / 9.9 user deals / 9.10 leaderboard):** DEFER.
  Activation gate: ≥25K monthly actives. Empty forum is worse than no forum.

- **Expert reviews voice:** CredStack Editorial. No fictional reviewers. Platform-voice
  reviews only. Real third-party reviewers added with attribution + their permission.

- **Debit + prepaid catalogue:** Quality over quantity. Better to ship 25 verified
  entries than 45 mediocre ones. Mark unverified rows with `data_pending: true` so the
  UI badges them.

- **Honest stats only.** No "100K+ users" placeholders. Verifiable numbers only —
  card count, data fields, last-verified date, ad-free positioning.

## Infrastructure

- **Cron scheduler:** Inngest.
  Reason: better retries, step durability, staggered scrape pattern already speced
  in `SCRAPING.md`. Vercel Cron is fallback for the simple daily reminder if Inngest
  setup is delayed.

- **Email transactional:** Resend.
  Reason: lowest-friction Indian-domain setup, generous free tier (3K emails/month).
  Backup: AWS SES if scale outgrows Resend.

- **AI provider:** Groq.
  Reason: lowest latency, cost. Llama 3.3 70B for reasoning, Llama 3.1 8B for fast.
  Backup: Vercel AI Gateway with multi-provider routing if Groq has outages.

- **Cache:** Upstash Redis.
  Reason: serverless-native, generous free tier (10K commands/day).

- **Scraping:** Firecrawl.
  Reason: handles JS-heavy bank pages; Redis-cached responses keep credits low.

## Deployment

- **Production target:** Vercel. Hobby tier initially, Pro at first scale signal.
- **Domain:** credstack.in (primary). credstack.app reserved as alternative.
- **Database:** Supabase. Free tier initially (500MB cap), Pro at first scale signal.
- **CDN / images:** Vercel default. No Cloudflare Pages or other.
- **Release schedule:** Production deploy AFTER Phase 1 ship gate met. No
  premature launch with placeholder content.

## Architecture rules (already in ROADMAP.md, restated)

1. **Money math is deterministic, never AI-generated.** Every rupee through code,
   not Groq. LLM is only for explanation layer.
2. **Cache before compute, compute before AI.** Redis > Postgres > deterministic
   function > LLM call.
3. **All money in integer rupees** (paise only for transaction-level).
4. **All new tables have RLS at creation.**
5. **No client-side use of service role keys.** Server actions or Edge Functions only.
6. **All AI-generated text labeled "Generated by AI"** with thumbs-up/down feedback.
